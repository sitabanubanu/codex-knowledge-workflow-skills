# Freshness Check

Use this when information can change.

## Mandatory Freshness Topics

Check dates for:

- latest/current/today/recent questions;
- prices, subscriptions, limits;
- laws, policy, school or exam rules;
- product features and AI model capabilities;
- news and public figures;
- schedules, releases, availability;
- security/privacy incidents;
- courses and learning resources.

## What To Check

- publication date;
- last updated date;
- effective date;
- product changelog or release notes;
- current official pricing or docs;
- whether old reviews describe an older product version.

## Handling Conflicts

If sources disagree:

1. prefer official or primary current source;
2. compare dates;
3. explain what changed;
4. mark old sources as stale when appropriate.

## Output

For substantial searches, include:

```text
Freshness:
- Current as of: ...
- Latest official source checked: ...
- Old/stale sources found: ...
- Remaining uncertainty: ...
```
