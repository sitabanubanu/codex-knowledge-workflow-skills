# Source Ledger

Use this to classify sources before trusting them.

## Source Types

| Type | Examples | Use |
|---|---|---|
| `Official` | homepage, docs, pricing, privacy, help center | product facts, limits, policies |
| `Primary` | law text, regulator notice, original paper, dataset, institutional announcement | highest authority for facts |
| `Professional` | specialist media, standards body, reputable expert review | context and independent analysis |
| `UserFeedback` | Reddit, Zhihu, forums, app stores, reviews | usability, reliability, pain points |
| `SEO/Affiliate` | best-of lists, sponsored pages, comparison blogs | discovery only; verify elsewhere |
| `RiskSource` | complaints, security reports, outage pages, refund posts | risk and caveat discovery |
| `Secondary` | summaries citing primary sources | quick orientation; verify primary |

## Authority Order

Prefer:

1. user constraints and current task;
2. official/primary sources;
3. specialist sources with named methods;
4. independent user feedback patterns;
5. SEO and affiliate sources only as leads.

## Rules

- Do not treat SEO listicles as neutral evidence.
- Do not treat one complaint as decisive unless it matches a broader pattern or high-severity risk.
- Do not trust copied summaries when no primary source is available.
- Do not mix official feature claims with user experience as if they answer the same question.
