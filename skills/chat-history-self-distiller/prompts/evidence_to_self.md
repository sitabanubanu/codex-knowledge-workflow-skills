# Evidence To Self Memory

Turn sampled chat evidence into `self.md`.

Use only evidence-backed claims. If evidence is weak, label it as weak rather than making it sound certain.

Before writing `self.md`, read `_analysis/core_thread_burn.md` and check `Burn Result`:

- `Core Thread Found`: use the thread as an organizing hypothesis, not as proof.
- `Weak Thread`: mention it only as a possible interpretation; organize `self.md` around dated scenes and confirmed patterns.
- `No Stable Thread`: do not use a unifying life story. Write a narrower evidence-backed memory file.

If evidence resists the thread, preserve the resistance instead of forcing it into the story.

Required sections:

1. Core identity and context
2. Values and priorities
3. Habits and rhythms
4. Important memories and repeated scenes
5. Relationship patterns
6. Growth trajectory
7. Tensions and unresolved themes
8. Correction log

For each important claim include:

- confidence: High / Medium / Low / Insufficient
- delivery tier: Confirmed Pattern / Probable Pattern / Hypothesis
- evidence ids or dated quotes
- evidence type: Direct Quote / Keyword Cross Validation / Sequence Pattern / Statistical Pattern / Human Inference

Avoid diagnosis. Avoid flattering or insulting labels. Prefer specific behavioral descriptions.

Rules:

- Core labels must be expanded into a mechanism chain: Trigger -> Processing move -> Output -> Boundary / failure mode.
- Test each core claim against the "100 random people" check. If it could apply broadly, make it more specific or demote it.
- Include concrete artifacts, dates, or scenes for core claims.
- State where the pattern does not apply.
- Confirmed Pattern may enter the main self memory.
- Probable Pattern may enter the main file only with a confirmation caveat.
- Hypothesis belongs in the correction/preview area or `evidence.md`, not as stable memory.
- Use `_analysis/behavior_patterns.json` when available for defenses, tension, silence, rationalization, or emotion-to-analysis claims.
- Use `_analysis/principle_statements.json` for worldview, rules, causal models, and belief reversals.
- Use `_analysis/cognitive_break_windows.json` for candidate turning points, but call them "candidate windows" unless direct evidence and user confirmation support stronger wording.
- Do not convert principle statements directly into values, or cognitive windows directly into transformations, without quote-level support and a stated alternative explanation.
- Do not show detector pattern names, regex templates, or raw counts in `self.md`. Translate internal signals into plain memory claims supported by dated evidence.
