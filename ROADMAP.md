# Roadmap

## v0.7 - Independent Acquisition (implemented)

- Replace the intermediary acquisition runtime with a project-owned Provider registry.
- Add `acquire-source-material` and `kw source doctor|matrix|plan|import`.
- Keep Bundle v2, SourceStatus, ASR, evidence audit and receipts stable.
- Restore `web-intent-scout` and `knowledge-learning-article` to the complete repository and sync set.
- Keep browser host declaration as a contract while leaving the independent browser identity project outside the workflow.
- Preserve a remote backup branch and local archive before migration.

## v0.8 - Source-wide naming and learning CLI

- Migrate internal `10_video` compatibility naming toward `10_source` without breaking receipts.
- Add a first-class `kw learn` wrapper around the current learning pipeline.
- Keep `video_analysis_pack.md` readable until downstream consumers migrate.
- Add direct RSS/V2EX adapters only when they preserve the existing Bundle boundary.

## v0.9 - Broader provider acceptance

- Add live, authorized fixtures for each structured Provider and operation.
- Measure availability and latency separately from evidence correctness.
- Extend provider contracts without embedding login secrets or platform-specific state in the evidence layer.

## v1.0 - Release-grade evaluation

- Complete at least 60 material-insufficient cases with zero false full-report releases.
- Run ordinary-agent comparison under the same inputs, tools and evidence rubric.
- Add human blind review and learning-effect evaluation.
- Publish only claims supported by reproducible evaluation artifacts.

The permanent product promise remains: no target-compatible primary material, no normal full report.
