# Conflict Resolution

Use this when sources disagree.

## Core Rule

Do not average conflicting sources. Decide which claim is stronger, or mark the conflict unresolved.

## Conflict Types

- price or plan limits differ;
- product feature claims differ;
- current status differs;
- legal/policy summaries differ from primary text;
- user feedback contradicts vendor marketing;
- old reviews describe a replaced version;
- different regions have different availability or rules.

## Resolution Order

Prefer:

1. official or primary source;
2. current effective date/current version;
3. source closest to the event, product, or policy;
4. transparent method or original data;
5. repeated independent confirmation;
6. user feedback patterns for lived experience.

## Special Cases

For product claims:

- official docs decide feature existence;
- repeated user feedback decides practical reliability;
- pricing pages decide current price;
- terms/privacy pages decide data handling.

For policy/legal claims:

- primary text beats summaries;
- jurisdiction and effective date are mandatory;
- if the primary text is unclear, say so.

For news/events:

- timeline matters;
- later corrections can supersede early reports;
- primary statements can prove what was said, not necessarily what is true.

## Output

Use this shape:

```text
Conflict:
Source A says:
Source B says:
Resolution:
Confidence:
Why:
```

If unresolved:

```text
Resolution: unresolved
Practical effect: ...
What would settle it: ...
```
