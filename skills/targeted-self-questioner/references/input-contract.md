# Input Contract

Use this file before starting any targeted-questioning workflow.

## Preferred Input

Prefer a handoff generated by an upstream analysis skill:

```text
_exports/10_targeted_questioner_handoff.md
```

This file should already contain identity locks, seed claims, follow-up leverage points, report routing hints, and compatibility boundaries.

## Supported Inputs

Use the strongest available source set:

1. Handoff file plus upstream report folder.
2. Upstream report folder without handoff.
3. Pasted report text plus user corrections.
4. User self-description only.

Confidence decreases as the input moves down this list.

## Files To Prefer

When reading a `chat-history-self-distiller` run folder, prefer:

```text
_exports/10_targeted_questioner_handoff.md
_analysis/identity_lock.md
_analysis/core_thread_burn.md
_evidence/evidence_ledger.json
_analysis/candidate_answers.json
_review/preview.md
_exports/00_overview.md
_exports/08_user_questions_and_evidence.md
_exports/99_corrections_and_review.md
_manifest.json
```

Do not read raw large chat files. This skill is not a raw evidence analyzer.

## Upstream Lock Inheritance

If upstream files exist, inherit:

- target person;
- canonical sender;
- human participant count;
- confirmed aliases and user-provided aliases;
- excluded system/non-human buckets;
- gender/pronoun source;
- mental-health boundary;
- third-party privacy boundary.

Do not reinterpret identity unless the user explicitly asks for an identity audit.

## Missing Upstream Locks

If no identity lock or evidence ledger exists:

- continue only as a weak-input calibration;
- label claims as `User-Provided` or `Report-Derived`;
- avoid stable identity claims;
- ask for the missing upstream folder if precision matters.

## Source Labels

Tag all claims with source type:

- `Evidence`: dated quote, evidence ledger, or structured upstream claim.
- `Inference`: model interpretation from evidence.
- `User Correction`: direct user answer or objection.
- `User Self-Report`: user-provided self-description not independently verified.
- `Weak Prior Report`: prior report without evidence pointers.
