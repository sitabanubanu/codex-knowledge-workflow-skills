# Claim Check

Use this before recommending a product, service, course, website, or tool.

## Core Rule

Recommendation requires claim checks.

Do not trust marketing claims, listicle rankings, or generated summaries without checking stronger evidence.

## Status Labels

| Status | Meaning |
|---|---|
| `Supported` | direct evidence supports the claim |
| `PartiallySupported` | part of the claim is supported with limits |
| `ReasonableInference` | cautious inference from evidence |
| `Ambiguous` | credible sources conflict |
| `Overstated` | claim is broader than evidence |
| `Unsupported` | checked evidence does not support it |
| `StaleOrSuperseded` | old evidence conflicts with current state |
| `Unverified` | not enough evidence checked |
| `Opinion` | recommendation judgment, not fact |

## Claims To Check

For tools/products:

- supported formats and platforms;
- free plan and paid limits;
- privacy/local/cloud data flow;
- export ability and lock-in;
- Chinese support or regional access;
- API/model dependency;
- cancellation/refund process;
- uptime or maintenance if relevant.

For courses:

- instructor identity;
- curriculum;
- price/refund;
- student outcomes evidence;
- certificate value;
- update date.

For policy/fact searches:

- official source;
- effective date;
- jurisdiction;
- whether summaries match the primary text.
