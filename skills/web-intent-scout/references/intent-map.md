# Intent Map

Use this before searching broad or ambiguous requests.

## Core Rule

Search intent before search terms.

The user's literal words are often only one surface form of the real decision.

## Intent Table

Build this compact table when ambiguity matters:

| Term | Possible meaning | User signals | Search vocabulary | Needed source type |
|---|---|---|---|---|

Examples:

| User term | Possible meanings |
|---|---|
| "AI writing tool" | academic polishing, copywriting, citation support, paraphrasing, detection evasion, local editor |
| "document analysis" | Q&A over files, summary, extraction, legal review, multi-document synthesis, OCR |
| "course" | free tutorial, paid bootcamp, university course, certificate, exam prep |
| "reliable website" | official source, low-bias information, safe shopping, privacy-respecting service |
| "latest policy" | official law/regulation, institutional notice, news explainer, implementation date |

## Direction Questions

Ask up to 3 questions only when a branch choice would materially change the search.

Prioritize:

- task: buy, use, learn, verify, compare, cite, troubleshoot;
- scope: product, policy, institution, course, news, guide, official source;
- constraints: price, region, language, privacy, local/cloud, platform;
- evidence need: official proof, user reviews, independent tests, legal/medical/financial authority.

If the user says to proceed without questions, write assumptions and search.

## Search Assumptions

Use this when skipping clarification:

```text
Search Assumptions:
- I will treat the request as ...
- I will include/exclude ...
- I will prioritize ...
```
